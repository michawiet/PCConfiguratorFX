Part Selection Summary – programs main screen

1.       This element of the windows title will always show the current window/screen that is being displayed – it will be different in every of the part selection screens and also in help windows.

2.       This is where help files can be found and accessed. Screenshot on the right shows expanded menu after clicking on this element. How to use is this help element. FAQ and Terminologies is a really useful compendium of knowledge about computer parts.

3.       This group of elements contains the part type names. There is a ability to hover with mouse cursor on one of the names to get tooltip with a short description of what is this part. On the right there is a screenshot of CPU tooltip.

4.       Another group of elements which this time show the actual images of the parts. When no products is selected a iconographic placeholder is being displayed, that closely matches a real part. As with the item group before, there is a ability to hover over this elements. The effect of that action is a enlarged image of the product, that  shown again on the right.

5.       Third group of elements in this screen contains the names of the products with few basic parameters that are unique to the type of the part and also the placeholders (if no part of this category was selected) for products with disabled (greyed out text) effect clearly indicating that no part was selected.

6.       Second to last group of elements is a column where prices of selected products are shown – obviously if no product of given category was selected placeholder will be displayed (a greyed out “-“ sign).

7.       Last but not least group of elements is filled buttons, which upon action of a click will open a new scene of selecting a product. Example of such a scene is  shown below.

8.       This element displays the cumulative price of all selected parts.

9.       Pie chart element initially displays only the legend and automatically updates with the selection of new parts. It visually shows how much money each of the computer components eats up. General advice is to keep computing parts of the computer with the biggest area in this chart and don’t spend too much on the case (unless the visual aspect is important).

10.   Stacked bar chart shows an approximation of performance in few of the typical workloads. Stacked here is crucial – it can help with seeing how much each part of the computer is affecting overall performance. For more information how to choose each part “FAQ and Terminologies” help section should be checked out.

 

Example part selection screen

1.       First to note is the updated title, that now indicates that program is in part selection screen.

2.       This element is the button, that is initially disabled when no product is selected. After clicking the active button selected product will be added and “Selection part summary” screen will be shown.

3.       This element contains the names of the columns. All of the columns can be rearranged based on user needs. All of the columns with text or number content can be sorted by simply clicking on the desired column label. Example of sorting order indicator was marked with a red square (result of clicking 2 times on the columns)

4.       Those buttons will disable the currently applied filters (Reset filters) and apply the selected filters respectively (Apply filters).

5.       Check box type filter – clicking on all changes the enabled/disabled state of the all the checkboxes to disabled/enabled. Filter used with repeatable text values.

6.       Number text filed filter – filter that accepts only number values with floating point separator. It is used as a number range filter, where numbers do not repeat a lot that would justify usage of filter #7.

7.       This filter upon clicking will be expanded to allow selection of a value from a range. Left element will not display higher values than the selected value in the right filter element (and logically in the other direction).

8.       A selected row – row can be selected by clicking on a visible one. Blue colour indicates that the row was selected. Selection will perish if applied filters will not allow previously selected element to be visible anymore.
